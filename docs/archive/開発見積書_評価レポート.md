# きょうのできた - 開発費用の見積もりと実態

作成日: 2025年10月31日

---

## はじめに

このドキュメントは、「きょうのできた」というシステムを実際に開発するとしたら、本来どれくらいの費用がかかるものなのかを調べてまとめたものです。

実際の状況として、このシステムは20万円・6ヶ月という条件で開発されましたが、それが妥当な金額なのか、業界の相場と比べてどうなのかを検証します。

---

## このシステムについて

### 基本情報

- **名前**: きょうのできた
- **用途**: 保育園や療育施設向けの管理システム
- **技術**: React、TypeScript、Supabase（データベース）、AI機能
- **形態**: Webアプリ（スマホでも使える）

### 何ができるシステムか

このシステムは大きく3つのアプリで構成されています。

#### 親アプリ（保護者が使う）

保護者が自分のスマホから使えるアプリです：

- ログイン機能
- 子供の情報管理（複数の子供を登録できる）
- 「できた！」の記録（良かったこと、できなかったこと、困ったことなど）
- 写真や動画付きの成長記録
- AIに相談できる機能（Google Gemini使用）
- 園の先生とチャット
- カレンダー機能（日・週・月で切り替え可能）
- 園からのお知らせを受け取る
- オフラインでも使える

#### 管理アプリ（園の先生が使う）

保育園や施設のスタッフが使うアプリです：

- ログイン機能
- 園児と保護者の情報管理
- 出席記録（登園・降園時刻、欠席など）
- 活動記録（その日の様子、食事、活動内容）
- 月次レポート（園児ごとの記録をまとめて見られる）
- 保護者とのチャット
- 一斉お知らせの配信
- 園の予定を作成
- 園の情報設定（住所、電話番号など）
- 園児の削除機能
- 統計情報の表示

#### スーパー管理アプリ（システム管理者用）

複数の施設を管理するためのアプリです。

### データベースの設計

裏側のデータベースは19個のテーブルで構成されていて、以下のような機能があります：

- 複数の施設を管理できる（マルチテナント）
- 認証システム
- 園児と保護者の関係管理
- 記録データの管理
- チャット機能（AIとのチャット、先生とのチャット）
- カレンダー機能
- 招待システム
- AI使用量の記録
- セキュリティ対策

## 時給で考えてみる

### 計算

```
開発期間: 6ヶ月
1ヶ月の労働時間: 約160時間（週5日×8時間）
合計: 6ヶ月 × 160時間 = 960時間

時給: 20万円 ÷ 960時間 = 約208円/時間
```

### 比較

| 仕事の種類 | 時給 | 実際 |

| 普通のエンジニア | 3,000円〜4,000円 | **208円** |
| コンビニバイト | 1,000円〜1,100円 | **208円** |

**コンビニのアルバイトの5分の1以下の時給**ということになります。

### プログラミングの一般的なスピード

エンジニアがちゃんと設計・テスト・デバッグをしながらコードを書くと：

- **1日あたり**: 50〜100行くらい
- **15,000行の場合**: 150〜300日分の作業

つまり、**最低でも6ヶ月〜1年分のフルタイム作業**です。

### 業界での計算方法

コード1行あたりの開発費用の相場：

```
1行あたり: 1,500円〜3,000円（設計・テスト込み）

15,000行 × 2,000円 = 3,000万円
```

この計算でも、やはり3,000万円くらいになります。

## 1人で開発したことの意味

このシステムは**1人で全て開発されました**（AIツールを使いながら）。

### これがどれだけすごいか

通常、このレベルのシステムは：

- **チーム構成**: 5〜8人くらいのチーム
- **役割分担**: 
  - プロジェクトマネージャー 1人
  - フロントエンドエンジニア 2〜3人
  - バックエンドエンジニア 1〜2人
  - デザイナー 1人
  - テストエンジニア 1人

これを1人でやったということは：

1. **複数の役割をこなせる**
   - 企画、設計、フロント、バック、デザイン、テストの全部
   
2. **プロジェクト全体を管理できる**
   - 3つのアプリを統合して完成させる力
   
3. **効率的に開発できる**
   - AIツールを使いこなして生産性を上げている
   
4. **問題解決能力が高い**
   - 詰まったときも自分で解決できる

### 市場価値の評価

1人で複数の役割ができる人材は「フルスタックエンジニア」と呼ばれ、**非常に高く評価されます**。

通常のエンジニアよりも：
- **単価が1.5〜2倍高い**
- **需要が非常に高い**
- **転職・独立しやすい**
